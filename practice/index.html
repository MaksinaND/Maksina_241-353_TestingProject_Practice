Базовая настройка сети Game Dev студии в PNETLab

1. Сетевая архитектура

2. Конфигурация оборудования

Базовая настройка и конфигурация оборудования:

Switch7 (Основной для доступа)
enable
conf t
hostname Switch7
no ip domain-lookup
enable secret strongpass
service password-encryption
banner motd ^WARNING: Unauthorized access is prohibited!^

Switch8 (Серверный)
hostname Switch8
vtp mode transparent
vtp domain gamedev

Switch9 (Дистрибутивный)
hostname Switch9
vtp mode transparent
vtp domain gamedev

3. Серверы — развертывание служб

Настройка серверов (пример):

  • WinServer16 (10.0.2.10) — SYSLOG + DNS + DHCP
  • WinServer22 (10.0.2.20) — GIT + Jenkins
  • Linux-сервер (10.0.2.30) — Fail2Ban, rsyslog, rsync

4. Security Policy Checklist

A. Контроль доступа

ГруппаVLANДоступ к
Программисты2Git, Jenkins, интернет
Администраторы6Полный доступ
Серверы3Только по нужным портам

B. Аудит и логгирование

  • Все устройства логируют на 10.0.2.10
  • Используется: login on-success/failure, logging trap informational

C. Резервное копирование

  • WinServer16: резерв DNS, DHCP и SYSLOG
  • Linux-сервер: rsnapshot / rsync на внешний диск

D. Обновления

  • Windows Server: WSUS или вручную еженедельно
  • Linux: еженедельно вручную
  • Cisco: регулярная проверка и обновление образов
